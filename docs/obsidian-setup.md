# Obsidian Setup

> How to use labs-wiki as an Obsidian vault for visual graph navigation.

## Opening as a Vault

labs-wiki is compatible with [Obsidian](https://obsidian.md/) out of the box — the `[[wikilink]]` format and markdown structure are native to Obsidian.

### Steps

1. Open Obsidian
2. Click "Open folder as vault"
3. Select the `labs-wiki/` directory
4. Obsidian will index all markdown files

### Recommended Folder Config

In Obsidian Settings → Files & Links:
- **New file location:** `wiki/concepts/` (default for new notes)
- **Attachment folder path:** `raw/assets/` (binary files go here)
- **Default path for new notes:** Select "In the folder specified below"

## Recommended Plugins

### Core Plugins (Built-in)

| Plugin | Why | Setting |
|--------|-----|---------|
| **Graph view** | Visualize `[[wikilink]]` connections | Enable, set colors by folder |
| **Backlinks** | See which pages reference the current one | Enable |
| **Tags** | Navigate by `tags:` from frontmatter | Enable |
| **Templates** | Use `templates/` for new pages | Set template folder to `templates/` |

### Community Plugins

| Plugin | Why | Install |
|--------|-----|---------|
| **Dataview** | Query pages by frontmatter (type, tier, score) | Community plugins → Browse |
| **Templater** | Advanced template logic | Community plugins → Browse |

### Dataview Examples

Once Dataview is installed, create a note with these queries:

**All stale pages:**
```dataview
TABLE quality_score, last_verified, tier
FROM "wiki"
WHERE date(last_verified) < date(today) - dur(90 days)
SORT quality_score ASC
```

**Pages by tier:**
```dataview
TABLE type, quality_score, last_verified
FROM "wiki"
WHERE tier = "core"
SORT quality_score DESC
```

**Low quality pages:**
```dataview
TABLE type, quality_score, sources
FROM "wiki"
WHERE quality_score < 50
SORT quality_score ASC
```

## Graph View Configuration

For the best graph experience, set node colors by folder:

| Folder | Color | Meaning |
|--------|-------|---------|
| `wiki/concepts/` | 🔵 Blue | Concepts and techniques |
| `wiki/entities/` | 🟢 Green | Tools, people, orgs |
| `wiki/sources/` | 🟡 Yellow | Source summaries |
| `wiki/synthesis/` | 🟣 Purple | Cross-cutting analysis |
| `raw/` | ⚪ Gray | Raw sources (less prominent) |

## Important Notes

- **Don't edit `wiki/index.md` manually** — it's auto-generated by scripts and skills
- **Don't edit `wiki/log.md` manually** — it's append-only by skills
- **Don't edit files in `raw/`** — they are immutable source documents
- **Do use Obsidian to browse and read** — it's excellent for navigation
- **Do create new pages via templates** — ensures correct frontmatter
