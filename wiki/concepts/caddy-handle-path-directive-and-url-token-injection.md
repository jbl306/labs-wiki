# Caddy handle_path Directive and URL Token Injection

## Overview

The Caddy web server's `handle_path` directive strips matched path prefixes before proxying requests to upstream services. This behavior can cause issues when backend applications build absolute URLs expecting the prefix to be present, such as token-based gating in KnightCrawler/Stremio addons. The solution is to inject the stripped token prefix back into URLs to maintain correct access control.

## How It Works

Caddy's `handle_path` directive matches a URL path prefix and removes it before forwarding the request to the backend. For example, a request to `/token1234/api` with `handle_path /token1234` will be proxied as `/api`.

KnightCrawler addons build absolute URLs using the request's `Host` header, which does not include the token prefix. This mismatch causes 404 errors because the backend expects the token prefix for authorization.

To fix this:

1. The token prefix (`KC_ACCESS_TOKEN`) is passed as an environment variable to the addon container.

2. A patch script (`patch-https-host.js`) modifies the addon to inject the token prefix back into URLs it builds, effectively reconstructing the original request path.

3. This ensures that URLs served by the addon include the token prefix, matching the expected gated paths.

This approach preserves the security gating mechanism while allowing Caddy to handle path routing cleanly.

## Key Properties

- **handle_path Behavior:** Strips matched path prefix before proxying, altering the request path seen by backend.
- **Token Injection:** Reinserts the stripped token prefix into absolute URLs generated by backend addons.
- **Environment Variable Usage:** Passes `KC_ACCESS_TOKEN` to backend containers for use in URL rewriting.

## Limitations

This fix requires modifying backend addon code or patching URL construction, which may not be possible for all applications. It assumes the token prefix is static and known at container startup. Dynamic tokens or complex routing may require more advanced proxy configurations.

## Example

Patch snippet injecting token prefix:

```js
const token = process.env.KC_ACCESS_TOKEN;
function injectToken(url) {
  if (!url.startsWith(`/${token}`)) {
    return `/${token}${url}`;
  }
  return url;
}
// Usage in addon URL generation
const proxiedUrl = injectToken(originalUrl);
```

Caddy config snippet:

```
handle_path /token1234 {
  reverse_proxy backend:port
}
```
