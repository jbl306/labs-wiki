# Labs-Wiki Architecture

## Overview

Labs-wiki is a personal knowledge wiki system powered by large language models (LLMs) that auto-ingests diverse source materials such as URLs, papers, and repositories, compiling them into structured wiki pages. Its architecture is designed for durability, extensibility, and integration with AI agents to facilitate knowledge management and retrieval.

## How It Works

Labs-wiki employs a three-layer architecture:

1. **Raw Layer (`raw/`)**: This layer stores immutable source files, including raw documents and metadata, ingested from various inputs such as URLs or shared files. These raw files are minimal initially and are expanded by the auto-ingest pipeline.

2. **Wiki Layer (`wiki/`)**: This layer contains compiled wiki pages organized into subfolders for concepts, entities, sources, and synthesis pages. The pages are generated by processing raw sources using GPT-4.1 via the GitHub Models API. The wiki layer supports cross-referencing and maintains metadata such as page tier and quality scores.

3. **Agent Schema (`AGENTS.md`)**: This schema defines the structure and interfaces for AI agents that interact with the wiki, enabling tools like search, read, and list operations via a Model-Context-Protocol (MCP) server.

The system includes a Docker sidecar that watches the raw directory for new or pending files, triggering the auto-ingest pipeline. Smart URL handlers are implemented to fetch content from sources like Twitter, GitHub, and HTML pages with vision capabilities. The MCP server (`wiki_mcp_server.py`) exposes APIs for agent interactions.

Labs-wiki maintains an index catalog of wiki pages and supports quality scoring and tiering, although scoring is not fully implemented yet. The architecture is designed to integrate graph-based features as an additional 'Layer 2.5' derived from the wiki pages themselves.

## Key Properties

- **Layered Architecture:** Separation of raw immutable sources, compiled wiki pages, and agent schemas enables modularity and durability.
- **Auto-Ingest Pipeline:** Automated ingestion and processing of raw sources using GPT-4.1 with smart URL handlers.
- **MCP Server:** Provides standardized tools for wiki search, reading, and listing to AI agents.
- **Cross-Referencing:** Wiki pages link to each other via wikilinks, supporting rich navigation.
- **Quality Metadata:** Pages have tier and quality_score metadata for future ranking and promotion.

## Limitations

Current quality scoring and promotion mechanisms are not yet implemented, limiting automated content curation. The system relies on external LLM APIs which may introduce latency and cost. The architecture currently lacks integrated graph analysis features, which are planned for future integration.

## Example

When a new raw source file is placed in the `raw/` directory (e.g., a shared arXiv paper URL), the Docker sidecar detects it and triggers the auto_ingest.py script. This script fetches the full content, processes it with GPT-4.1 to extract concepts, entities, and source pages, and writes compiled markdown pages into the `wiki/` directory. The MCP server then allows AI agents to query these pages.

## Practical Applications

Labs-wiki serves as a personal knowledge management system that leverages LLMs to automatically ingest, summarize, and interlink knowledge from diverse sources. It supports research workflows, software documentation, and AI agent integration for enhanced knowledge retrieval.
