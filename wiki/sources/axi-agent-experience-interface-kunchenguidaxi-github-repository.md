# AXI: Agent eXperience Interface (kunchenguid/axi GitHub Repository)

## Summary

AXI (Agent eXperience Interface) is a new paradigm for building agent-native CLI tools, defined by 10 ergonomic principles that optimize for token efficiency and agent usability. The repository provides benchmarks demonstrating AXI's superior performance and cost-effectiveness compared to traditional CLIs and structured tool protocols like MCP. It includes reference implementations, benchmarking harnesses, and a skill definition for building AXI-compliant tools.

## Key Points

- AXI introduces 10 design principles for agent-ergonomic CLI tool development, prioritizing token efficiency and agent actionability.
- Benchmarks show AXI-based tools outperform both traditional CLI and MCP-based interfaces in success rate, cost, and interaction efficiency.
- The repository provides implementation scaffolding, benchmarking harnesses, and a skill definition to guide developers in building AXI-compliant tools.

## Concepts Extracted

- **[[AXI Design Principles for Agent-Ergonomic CLI Tools]]** — The AXI Design Principles are a set of 10 ergonomic standards for building CLI tools that AI agents can use efficiently via shell execution. These principles focus on minimizing token usage, maximizing agent clarity, and ensuring robust, actionable interactions, setting a new standard for agent-native tool design.
- **[[TOON (Token-Oriented Object Notation) Format]]** — TOON is a serialization format designed to minimize token usage for LLM agents while retaining structure and readability. It is a core requirement for AXI-compliant tools, offering significant token savings compared to JSON.
