## Technical Details

- Python is externally-managed on Ubuntu 24.04 (PEP 668) — cannot use `pip install` directly
- Used `pipx install mempalace` which creates isolated venv automatically
- `mempalace init <dir> --yes` scans project for entities, creates `mempalace.yaml` and `entities.json` in project dir
- `mempalace mine <dir>` reads mempalace.yaml from project dir, mines files into palace at `~/.mempalace/palace/`
- ChromaDB embedding model (all-MiniLM-L6-v2, 79.3MB) auto-downloads on first mine
- No `--version` flag; use `mempalace status` or `mempalace -h` to verify ### MemPalace Architecture (key details for implementation)
- Palace data at `~/.mempalace/palace/` (ChromaDB persistent storage)
- Config at `~/.mempalace/config.json`
- Identity at `~/.mempalace/identity.txt` (L0 layer, ~100 tokens, always loaded)
- MCP server: `mempalace mcp` (stdio transport) — 19 tools
- No Docker required — fully local, no API keys for embeddings
- Collection name: `mempalace_drawers` ### OpenMemory Current State
- **Containers are NOT running** (docker ps shows nothing for memory/qdrant/openmemory)
- But config files exist: `config/openmemory/config.json`, `config/openmemory/.env`
- 55 memories stored (retrieved via MCP list_memories while service was apparently running earlier or cached)
- Categories: NBA-ML (6), KnightCrawler (7), Wiki (8), Grafana (8), Docker (6), GitHub Models (5), HTTP Shortcuts (3), Other (7)
- One memory contains Grafana admin credentials (sensitive — handle carefully during migration) ### Homelab Infrastructure Patterns
- Compose files in `compose/` dir, included from `compose/docker-compose.yml`
- Config at `config/<service>/`, data at `data/<service>/`
- Deploy order in `deploy.sh`: proxy, tunnel, infra, media, stremio, riven, cloud, photos, nba-ml, memory, web, monitoring, jobs
- `setup.sh` creates dirs for each service including `openmemory` and Qdrant data dirs
- `backup.sh` has Qdrant snapshot backup (step 7/8)
- `.env.example` has OPENMEMORY section at bottom
- `~/.copilot/mcp-config.json` has two servers: openmemory (SSE, port 8765) and labs-wiki (stdio)
- Compose validation: `docker compose -f compose/docker-compose.yml --env-file .env config > /dev/null` ### MCP Config Format ```json { "mcpServers": { "openmemory": { "type": "sse", "url": "http://localhost:8765/mcp/copilot/sse/jbl" }, "labs-wiki": { "type": "stdio", "command": "python3", "args": ["/home/jbl/projects/labs-wiki/scripts/wiki_mcp_server.py"] } } } ``` MemPalace MCP will use stdio transport: `mempalace mcp` command. ### Key Decisions
